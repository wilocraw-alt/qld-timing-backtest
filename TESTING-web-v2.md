# 웹 v2 기능 테스트 가이드 (2026-06-25)

라이브 PWA(`web/template_index.html`)에 추가된 3가지 기능을 직접 확인하는 방법.

## 새로 추가된 기능

- 기능1 — 거래기록 티커 입력이 SOXL/QLD 드롭다운(select)으로 변경
- 기능2 — 거래목록 UX 개선
  - 종목별 요약 강화(보유수량·평단가·평가손익)
  - 거래목록 최신 20개만 표시 + "더 보기/접기" 토글
  - 입력폼 접이식("+ 거래 추가" 토글)
  - 삭제 시 확인창
- 기능3 — "목표 배분" 카드에 현재 배분% + 리밸런싱 지시 추가
  - 현재 보유 비중(거래기록 수량 합산 기준)
  - 목표비율에 맞추려면 어느 종목을 얼마(매수/매도 $·주수) 조정해야 하는지
  - 신호 반영: ON=60/40, OFF=SOXL 0%·현금60%, NEUTRAL=보유상태 기반 동적

## 방법 A — 로컬에서 바로 보기 (추천, 가장 빠름)

이미 빌드된 `_site/`가 있으면 서버만 띄우면 됩니다.

```
python3 -m http.server 8000 --directory _site
```

- 브라우저에서 `http://localhost:8000` 접속.
- 최신 시세로 다시 빌드하려면(네트워크·의존성 필요):
  ```
  pip install pandas numpy yfinance pyyaml matplotlib pillow   # 최초 1회
  python web/build_site.py --out _site
  ```

## 방법 B — 라이브 사이트에 반영해서 보기

- 이 커밋을 push한 뒤, GitHub Actions의 `daily-site` 워크플로를 수동 실행하면 라이브 사이트가 갱신됩니다.
  - 사이트 상단 "⟳ 재계산" 링크 → GitHub Actions 페이지 → "Run workflow"
  - 또는 매일 07:30 KST 자동 빌드 대기.
- 라이브 URL: https://wilocraw-alt.github.io/qld-timing-backtest/

## 확인 체크리스트

- 기능1: "거래 기록" 카드에서 티커가 입력칸이 아니라 SOXL/QLD 선택 드롭다운인지.
- 기능2:
  - "+ 거래 추가"로 입력폼이 접혔다 펼쳐지는지.
  - 거래를 21건 이상 추가하면 "더 보기" 토글이 생기는지.
  - 종목 요약에 보유수량·평단·평가손익이 보이는지.
  - 삭제(✕) 누르면 확인창이 뜨는지.
- 기능3:
  - 거래를 **수량(qty) 포함**으로 추가하면 "목표 배분" 카드에 현재 비중%와 매수/매도 지시가 뜨는지.
    - 예: SOXL 10주 매수 + QLD 5주 매수 → 현재 비중·각 종목 매수/매도 $·주수 표시.
  - 수량을 비워 추가하면 해당 종목에 "수량 입력 필요" 경고가 뜨는지.

## 신호 상태(ON/OFF/NEUTRAL)별 리밸런싱 테스트 (로컬)

실제 신호는 라이브 시세에 따라 정해지므로, 다른 상태를 시험하려면 로컬에서 데이터 파일을 잠깐 바꿔 봅니다.

- `_site/data/latest.json` 의 `"signal": { ... "state": "ON" }` 값을 `"OFF"` 또는 `"NEUTRAL"` 로 바꾸고 새로고침.
  - OFF → SOXL 목표 0%, "SOXL 전량 매도 → 현금" 지시 확인.
  - NEUTRAL → SOXL을 이미 보유 중이면 60/40 유지, 미보유면 OFF목표로 동작(보유상태 기반).
- 테스트 후 원래 값으로 되돌리거나 다시 빌드하면 됩니다.

## 참고

- 거래기록은 브라우저 localStorage(`wath_journal_v1`)에 기기별 저장 — 서버 전송 없음.
- 자동검증 현황: 리밸런싱 계산 단위테스트 31/31 통과, 정적검증 11/11 통과.
  헤드리스 렌더 스모크 테스트는 별도 진행 중.
- 리밸런싱 계산 로직은 순수함수 `window.__computeRebalance(journal, prices, signalState)` 로 노출되어 있습니다.
