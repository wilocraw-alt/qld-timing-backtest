# selection.md — 방법론 선정 메타절차

## 요약 (사용자용)

8개 발굴 방법론 중 희망사항의 성격에 맞는 **2~3개를 고르는 절차**. 매니저가 단독으로 고르지 않고, 우선 발산(워커 추천)한 후 수렴(매니저 종합)한다. 고른 방법은 각각 **하나의 구체적 미지수**에 결박되어야 하며, 미지수 없는 방법은 탈락.

---

## 1. 목적별 분류 맵

방법론을 다섯 가지 카테고리로 분류:

| 카테고리 | 목적 | 방법론 |
|----------|------|--------|
| **Root/Purpose** | 근본 원인·동기 파악 | 5 Whys, First Principles |
| **Reframe/Problem** | 문제 재정의·새 관점 | How Might We |
| **Evolution/Creativity** | 변형·확장·조합 | SCAMPER, Crazy 8s, Analogies |
| **Value** | 사용자 가치·맥락 이해 | Jobs-to-be-Done |
| **De-risk** | 위험 식별·편향 해소 | Pre-mortem |

---

## 2. 모호도별 기본 추천 세트

(1차 안내용 — 최종 선택은 발산→수렴으로)

| 모호도 | 기본 추천 | 이유 |
|--------|-----------|------|
| **Vague** ("뭔가 ~했으면") | 5 Whys + First Principles | 방향 자체를 정해야 하므로 근본 탐색 |
| **Fuzzy** ("X를 하고 싶은데") | HMW + SCAMPER | 문제는 있으니 재구성·변형 확장 |
| **Clear** ("X를 Y로") | Pre-mortem + JTBD | 빠른 검증 + 사용자 가치 확인 |

Fast-track: Clear 수준이면 S2 발산 자체를 생략하고 곧장 S3 valuation.

---

## 3. 발산→수렴 메타절차

### S1-MetaDiverge (워커에게 위임)

매니저가 현재 희망사항 + 모호도 분류를 취합해 1~2명의 워커에게 PROPOSE:

```
희망사항: "<요약>"
모호도: Vague / Fuzzy / Clear

위 희망에 가장 적합한 방법론 2~3개를 아래 카테고리 밸런스를 고려해 추천하세요:
- Root/Purpose: 근본 원인 발굴 필요성?
- Reframe: 문제 재정의 필요성?
- Evolution: 변형·확장 필요성?
- Value: 사용자 가치 이해 필요성?
- De-risk: 위험 식별 필요성?

각 추천 방법에 대해:
1. 왜 이 방법인가 (희망 성격과의 정합)
2. 이 방법이 반드시 답해야 할 **하나의 미지수**
```

동시에 매니저도 자체안을 작성.

### S1-MetaConverge (매니저 수렴)

워커 응답 + 매니저 자체안을 종합해 아래 규칙으로 최종 선정:

### 규칙

1. **적용 방법은 최대 2~3개** (초과는 분산됨). 1개만 있어도 괜찮음.
2. **각 방법은 풀어야 할 미지수 1개에 결박** — "이 방법으로 무엇을 알게 될 것인가?"라는 질문에 구체적 답이 나와야 함. 미지수가 없으면 (이미 답을 알거나 중복이면) 탈락.
3. **카테고리 다양성** 선호 — 같은 카테고리 2개보다 다른 카테고리 2개가 더 넓은 커버리지.
4. **모호도 적합성** 우선 — Vague에 SCAMPER를 고르는 것은 가능하지만, Root/Purpose가 더 우선.
5. **사용자 확인** — 선정 결과를 사용자에게 제시 (Gate G1): "방법론 A로 <미지수1>를, 방법론 B로 <미지수2>를 해결하겠습니다."

---

## 4. 메타절차 요약

```
모호도 분류 → [발산] 워커에게 "적합 방법론 추천" PROPOSE
           → [수렴] 매니저가 종합 → 규칙 적용 → 2~3개 선정
           → [결박] 각 방법에 미지수 1개 할당
           → [게이트 G1] 사용자 승인
           → [실행] S2 아이디어 발산 진입
```
